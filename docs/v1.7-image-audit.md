## Current Directory Issues

- The runtime structure now follows product modules, but several approved directories are intentionally still empty: `archive/template`, `archive/frame`, `archive/cover`, `explore/featured`, `explore/archive`, `explore/inspiration/companion`, `ui/state`.
- A large batch of generated drafts was mixed into runtime folders before cleanup. These were moved into `_source/raw-imports/` so they are preserved but no longer treated as runtime assets.
- Legacy decorative duplicates existed in both `landing/decoration/` and a generic `decoration/` tree. The generic copies were preserved under `_source/legacy-duplicates/` instead of being deleted.
- Explore food and location assets mixed city cards, terrain cards, cover art, Chinese file names, and draft outputs in the same folders. These are now separated into `explore/cities/`, `explore/food/`, and `explore/inspiration/location/`.
- Shared state assets that were previously grouped under itinerary or workspace subtrees were normalized into `ui/empty/`, `ui/feedback/`, `ui/loading/`, `ui/weather/`, and `ui/transport/`.
