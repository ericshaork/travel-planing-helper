# v1.7 Phase 3.4 Archive Content Refinement

## Scope

This phase refines the remaining Archive content modules so they match the reading language established in Phase 3.3.

It does not change:

- Explore data structure
- API
- Supabase
- Explore flow
- Workspace
- image directories
- drawer interaction model

## Updated Components

### `ArchiveHeader`

Role:

- archive top label instead of generic functional header

Visual rules:

- archive number chip from `externalId` or fallback id
- source label based on `creatorType`
- city / days / trip type as small archive metadata chips
- light label note and bookmark accent

### `ArchiveHighlights`

Role:

- “highlight records” instead of generic chip list

Visual rules:

- each highlight becomes a small archive card
- numbered highlight cue
- optional theme / city tags
- stamp accent stays light and does not dominate the section

### `ArchiveFood`

Role:

- travel food notes instead of plain list rows

Visual rules:

- each food item becomes a food note card
- uses existing Explore food images when category suggests a match
- falls back to archive-safe food fallback image
- keeps reason and district as reading-first content

### `ArchivePOI`

Role:

- place records instead of generic POI list

Visual rules:

- numbered entries
- place name first
- reason as the core note
- type / district / stay-duration as secondary tags

### `ArchiveMap`

Role:

- route relation note instead of empty technical placeholder

Visual rules:

- route order is visible
- POI list becomes a route sequence
- still does not replace real interactive map logic

## Shared Visual Language

These modules now consistently use:

- `ArchivePaperPanel`
- light / warm paper layers
- bookmark accents
- label / tape / stamp accents

Usage rule:

- not every section uses every accent
- decoration is sparse and secondary to reading
