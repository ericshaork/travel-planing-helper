# v1.7 Image Quality Report

## 1. Audit Summary

本报告检查的是“当前图片资产体系是否足以支撑后续接图”。

当前分支审计结果：

- 未发现已提交的正式图片资源文件。
- 未发现已提交的运行时图片目录。
- 未发现代码中稳定的正式图片引用。

因此，本次质量结论不是“现有图片质量差”，而是“当前分支尚未形成正式图片资产层”。

## 2. Findings

### Q1. Duplicate Images

当前分支结论：

- 无法确认实际重复图片，因为未扫描到已提交图片文件。

潜在历史风险：

- 同一城市封面可能被同时存为 city / featured / cover 三份。
- 同一贴纸可能同时存在 cutout、sheet-cut、final 三种命名版本。

后续处理建议：

- 正式迁移前先建立资产台账。
- 用“内容主题 + 页面用途 + 是否原图”去重。

### Q2. Naming Issues

当前分支结论：

- 无法直接审计现有文件名，因为没有已提交图片文件。

已知高风险命名模式：

- `final.png`
- `final2.png`
- `01.png`
- `hero_new_new.png`
- 中文名带空格
- 多版本后缀混乱

建议规则：

- 全小写
- `kebab-case`
- 加上模块和用途

### Q3. Wrong Folder Placement

当前分支结论：

- 当前主要问题不是“放错”，而是“正式模块目录尚未建立”。

已知结构性问题：

- 旧分类思路按 `city / food / weather / transport / stickers / icons` 组织。
- 新的网站架构已经变成 `Landing / Explore / Archive / Workspace / Brand`。

影响：

- 同一种图无法快速找到是否服务某个页面。
- 资源准备会和产品结构脱节。

### Q4. Missing Transparent Background

当前分支结论：

- 无法逐个检查，但后续下列资源强依赖透明背景：
  - 贴纸
  - 章印
  - 胶带
  - 角标
  - 小型装饰
  - hover 态图标

建议：

- 成品装饰资源默认导出 `png/webp` 透明背景版。
- 不透明背景大图保留在 `_source/`。

### Q5. Sheets Mixed With Runtime Assets

当前分支结论：

- 当前分支未见实物，但这是最需要提前规避的问题之一。

风险：

- 设计 sheet、原始大图、裁切成品图混在一起，会让开发误用大图或误接错素材。

建议：

- 凡是整张贴纸板、大拼图、原始母版，全部进入 `images/_source/` 或 `images/stickers/sheet/`。
- 运行时目录只放成品图。

### Q6. UI Needed But Missing

当前分支结论：

以下是后续 UI 明确需要、但当前分支尚未具备的资源：

- Landing hero 主视觉
- Landing 背景纹理和装饰
- Explore Inspiration 四类卡片图
- Explore Featured 封面
- Explore Archive 列表封面
- Archive Viewer 主封面
- Archive 纸张纹理
- Workspace 地图空状态图
- Workspace 天气建议图
- 全站品牌 logo / favicon

## 3. Severity

### High

- 没有正式 runtime 图片系统
- 没有按产品模块组织
- 没有成品图与原图分层

### Medium

- 未来若直接迁入旧素材，极易出现命名混乱和目录错放
- Archive / Explore / Landing 缺少明确第一批资源清单

### Low

- 图标细分目录、hover 态、装饰类资源命名规范尚未落地

## 4. Recommended Next Step

建议后续执行顺序：

1. 先建立真实 `images/` 目录骨架
2. 再整理旧素材台账
3. 再按本轮迁移规则落位
4. 最后才接入页面

## 5. Final Conclusion

当前图片系统的核心问题不是“现有素材不够漂亮”，而是：

- 还没有和产品架构对齐的正式资产系统
- 还没有稳定可维护的运行时目录
- 还没有明确的缺图优先级

本阶段完成后，后续团队就可以在不改代码的前提下，先把图片准备工作按模块推进起来。
