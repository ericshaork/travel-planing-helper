# v1.7 UI Asset Binding Map

## Goal

This document maps current UI components to runtime image folders and distinguishes between:

- assets that already exist
- assets that are still missing
- components that already reference images in code
- components that are still image-unbound

## Key Finding

As of Phase 3.0, the `public/images` system is organized, but the new Explore, Archive, and Workspace pages do not yet bind those assets in runtime code.

Current code search result:

- `next/image` usage exists in `components/trips/SavedTripCard.tsx`
- that component uses `trip.cover_image_url`
- no current Explore / Archive / Workspace component directly references `public/images/...`

This means the current Phase 3 work is primarily a readiness and sequencing problem, not a refactor problem.
