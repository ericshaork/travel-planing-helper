# v1.7 Phase 3.3 Archive Visual Integration

## Scope

This phase adds the first static visual layer to the Travel Archive Viewer without changing:

- data structure
- API contracts
- Explore content schema
- drawer interaction model
- mobile / desktop archive routing model
- animation behavior

The goal is to make Archive feel like a readable travel archive instead of a plain stack of panels.

## Updated Components

### `ArchiveDrawerShell`

Visual rule:

- desktop drawer now uses `archive-template-main.png` as the shell background
- template is applied as a container layer only
- a soft overlay keeps text readable

### `ArchivePageShell`

Visual rule:

- mobile full-page archive now uses `archive-template-mobile.png`
- the template remains a background layer
- content keeps the same reading flow, only the shell mood changes

### `ArchiveViewerContent`

Visual rule:

- the main archive body now sits inside a shared paper-backed reading surface
- content order remains:
  - intro
  - highlights
  - itinerary
  - food / POI
  - map

### `ArchiveCover`

Visual rule:

- cover remains the first focal point
- uses:
  - archive template
  - archive paper
  - archive photo frame
  - city or archive cover candidates
- tags are now visible at the opening surface
- featured archives can receive a stamp-style accent

### `ArchiveIntro`

Visual rule:

- uses a light paper surface
- adds bookmark and label-note style accents
- featured reason becomes a lighter archive note block

### `ArchiveItinerary`

Visual rule:

- uses a warm paper surface
- adds bookmark and tape-style accents
- each day reads like a route note rather than a plain list card

### `CreateMyVersion`

Visual rule:

- CTA remains visually clear and structurally unchanged
- copy now emphasizes the Archive -> Create -> Workspace handoff
- a light label-note accent is added without changing button flow

## Archive Shell Rules

- template belongs to the outer container, not each content block
- paper belongs to readable sections
- frame belongs to the opening visual moment
- bookmark and decoration are optional accents, not layout drivers
- overlays must stay light enough to preserve text contrast

## Desktop / Mobile Difference

Desktop drawer:

- uses `archive-template-main.png`
- appears as a contained archive panel beside Explore

Mobile page:

- uses `archive-template-mobile.png`
- keeps full-page reading with the same archive component structure

## Not Included Yet

Still intentionally not done in this phase:

- drawer slide-in animation
- archive page-turn animation
- stamp animation
- advanced collage layout
- deeper cover composition logic
- Archive food / POI / map visual rework
