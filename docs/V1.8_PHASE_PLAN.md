# Wanderly v1.8 Phase Plan

## 总目标

在 `v1.7` 可运行基线上，逐页完成 Landing、Explore、Workspace、Settings 的技术补全、UI/交互精修、图片资产正式接入和跨页面状态收口，形成一版可用于桌面端展示与真实试用的产品。

## 全局实施原则

- Desktop First：桌面端为正式设计与验收目标，移动端只做不严重破版的回归保障。
- 不推翻 `v1.7`：优先复用现有 AI、Trip、地图、登录、保存与 Explore 基础能力。
- 不把未来方向写成当前阶段必做：地图高级增强、长期记忆、国际内容等只做边界预留。
- 页面不是孤立开发：每个阶段都必须同时考虑导航、数据和状态衔接。
- 图片资产是每页正式工作的一部分，不是实现后再补的装饰任务。
- 每个 Phase 完成后都需要用户确认后再进入下一阶段。

## 版本提交原则

- 每个 Phase 完成后需要截图、变更清单和用户人工确认。
- 用户未确认前：
  - 不 `commit`
  - 不 `push`
- Phase 完成不等于必须单独提交。
- 整个 `v1.8.1` 准备阶段完成后，再统一决定是否提交。
- 页面开发版本的具体提交时机，由用户在验收时决定。

## Phase 0：项目基线与全局设计

### 目标

- 完成 `v1.7` 收档后的 `v1.8` 范围冻结。
- 明确 Landing、Explore、Workspace、Settings 四页关系。
- 明确首次访问逻辑、中文规范、中国内容范围、Desktop First 和图片资产规则。
- 对 Landing 做只读审计，为后续阶段提供明确输入。

### 技术工作

- 核对 `v1.7` 可运行基线与当前路由结构。
- 明确首次访问状态、根路径跳转和 `/welcome` 的全局逻辑方案。
- 明确 Explore、Workspace、Settings 与现有登录/保存机制的关系。

### UI 工作

- 冻结四个核心页面的定位与主链路顺序。
- 冻结整站中文规范与功能性文案原则。

### 图片资产工作

- 明确各页面的图片资产工作流程。
- 确认国外素材不进入 `v1.8` 正式 Explore 页面。
- 对 Landing 当前素材做只读盘点，不在本阶段改图。

### 与其他页面的关系

- Phase 0 是所有后续页面阶段的共同约束来源。
- 后续任何页面阶段都不得绕开本阶段的首次访问、中文规范和内容范围结论。

### 明确不做什么

- 不改代码；
- 不改旧文档；
- 不提前进入任一页面实现；
- 不把未来增强能力塞进本阶段。

### 验收标准

- `PRD_v1.8` 和 `V1.8_PHASE_PLAN` 完成；
- 四页关系、首次访问逻辑、中文规范、内容范围、图片规则明确；
- 用户确认后才能进入页面实现阶段。

### 是否需要用户确认

- 需要。

### 是否允许 commit 和 push

- 不允许。

## Phase 1：Landing 技术补全

### 目标

- 让 Landing 真正成为“首次访问欢迎页”，而不是长期首页。
- 完成首次访问状态、根路径跳转、`/welcome` 路由和与 Explore/Create/登录态的技术衔接。

### 技术工作

- 首次访问状态记录；
- 根路径默认跳转逻辑；
- `/welcome` 路由；
- Explore 主入口与 Create 次入口；
- Header 与导航状态统一；
- 登录态与首次访问态关系；
- Hero 图片加载、桌面适配、错误与 fallback；
- 与深层链接共存，不强制跳回 Landing。

### UI 工作

- 不做最终视觉定稿；
- 先确保入口层级、可理解性和操作路径清楚。

### 图片资产工作

- 核对 Hero 主图加载和尺寸策略；
- 标记缺失、失真或不适合运行时的 Landing 资产；
- 为下阶段视觉精修准备候选清单。

### 与其他页面的关系

- 必须与 Explore 入口、Create 入口、登录返回、全局 Header 一起设计。

### 明确不做什么

- 不复制 Create 表单进 Landing；
- 不把 Landing 做成老用户首页；
- 不在本阶段解决完整视觉精修。

### 验收标准

- 根路径与 `/welcome` 逻辑正确；
- 首次访问记录时机正确；
- 已完成引导或已登录用户默认进入 Explore；
- 深链访问不被强制打断。

### 是否需要用户确认

- 需要。

### 是否允许 commit 和 push

- 不允许。

## Phase 3：Explore 技术补全

- My Trips 入口衔接。

### UI 工作

- 明确“搜索 / 灵感 / AI 创建 / 档案书架”的层级。

- 盘点 Explore 当前图片是否适配正式范围；
