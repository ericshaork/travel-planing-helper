# v1.7 Phase 3.2.1 Explore Visual Refinement

## Scope

This phase refines the Explore home page visual hierarchy without changing:

- Explore architecture
- data shape
- API contracts
- resolver logic
- image directories

The work stays within the current component tree and existing assets.

## Updated Components

### `components/explore/InspirationCard.tsx`

Changes:

- image remains the first visual anchor
- category name is promoted with larger type
- selected count moves into a small top-right chip
- action copy becomes the second hierarchy level
- explanatory text is reduced and becomes secondary

Result:

- the card reads like a travel entry point
- it feels less like a generic option card

### `components/explore/InspirationDeck.tsx`

Changes:

- each card now receives a short action-oriented label
- above-the-fold cards get higher image priority:
  - `location`
  - `food`

Result:

- the deck reads faster
- the first row feels more intentional

### `components/explore/ExploreCard.tsx`

Changes:

- supports three visual modes:
  - `default`
  - `hero`
  - `compact`
- cover image remains primary
- city / days / trip type are promoted into chips
- summary becomes supporting text
- highlights and tags are trimmed by emphasis level

Result:

- archive cards feel less like product tiles
- title and destination metadata lead the scan

### `components/explore/ExploreFeed.tsx`

Changes:

- featured mode now renders:
  - one main hero archive
  - remaining supporting archives in a compact row
- default feed keeps the existing grid
- first visible archive image gets priority

Result:

- featured content now has clear hierarchy
- Explore home no longer treats featured and feed as the same visual weight

## Image Display Rules

Shared runtime rules continue through `ResolvedImage`:

- `next/image`
- ordered fallback sources
- fixed aspect ratio wrappers
- `object-cover`
- limited `priority` only for above-the-fold images

Priority usage after this phase:

- `location` and `food` inspiration cards
- featured hero archive
- first card in the regular feed

## Information Hierarchy Shift

### Inspiration cards

Before:

- image
- title
- description
- count

After:

- image
- title
- action cue
- secondary description
- state chip

### Explore archive cards

Before:

- image
- title
- summary
- mixed metadata

After:

- image
- title
- city / days / type chips
- short summary
- selected highlights
- limited tags

## Outcome

The Explore home page now has a clearer first scan:

1. inspiration entry
2. featured archive
3. archive feed

without requiring new assets or layout re-architecture.
