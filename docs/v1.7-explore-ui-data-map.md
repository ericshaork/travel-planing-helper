# v1.7 Explore UI Data Map

## Purpose

This document maps Explore UI modules to the data they need, and highlights which future image resources should exist before the visual phase begins.

This phase only stabilizes the data contract. It does not implement visuals, images, or animation.

## Core Principle

Page component
↓
Required content fields
↓
Optional media fields

The goal is to avoid organizing images by "whatever assets already exist". We should instead define assets from page needs.

## Current Missing Resource List

These are not blockers for the data phase, but they are missing for the visual phase:

- archive cover images
- inspiration category cover images
- featured archive visual assets
- mobile/desktop archive crop rules
- fallback illustrations for empty featured and empty archive states
- optional icon set for food / POI / archive metadata
