# v1.7 Explore Data Gap Report

## Scope

This report is based on:

- current Explore TypeScript data model
- current importer contract
- sampled real JSON files in the repo root:
  - `beijing_couple_2d_001.json`
  - `beijing_couple_3d_007.json`
  - `beijing_couple_3d_010.json`
  - `beijing_couple_4d_004.json`
  - `beijing_family_3d_003.json`

The project previously imported about 200 Explore items. This report uses the sampled files as a representative shape check, not a full census of all records.

## Fields Confirmed Present In Sampled JSON

Stable fields already present:

- `id`
- `slug`
- `title`
- `summary`
- `city.code`
- `city.name`
- `city.region_name`
- `trip_type`
- `days`
- `budget.level`
- `budget.note`
- `pace`
- `tags`
- `image_prompt`
- `daily_itinerary`
- `pois`
- `food`

These are enough for:

- Explore feed basics
- archive title / summary / itinerary / POI / food
- current importer pipeline

## Fields Missing From Sampled JSON

Archive content fields not present in sampled files:

- `archive_intro`
- `featured`
- `featured_reason`

Content source fields not present:

- `creator_type`
- `creator_id`
- `creator`

Inspiration taxonomy fields not present:

- `terrain_tags`
- `cuisine_tags`
- `season_tags`
- `companion_tags`

Future interaction fields not present:

- `likes`
- `views`
- `saved_count`

Media fields currently missing or incomplete for UI phase:

- `coverImageUrl`
- any archive cover asset mapping

## Compatibility Status

Current importer and schema now accept the future fields as optional.

That means:

- old JSON remains valid
- old imported rows remain valid
- future pipeline JSON can begin adding these fields gradually

## Current Gaps That Still Need Future Work

### Data gaps

- no stable archive introduction text yet
- no explicit inspiration taxonomy in current production JSON
- no featured curation metadata in current production JSON
- no creator/source identity beyond the pipeline metadata
- no popularity counters

### Media gaps

- no cover image field populated in sampled JSON
- no asset mapping between archive records and image files
- no defined image folder contract for Explore UI modules

## Conclusion

The current 200-item Explore content set is structurally good enough for:

- list browsing
- archive reading
- Create My Version handoff

It is not yet rich enough for the next visual/content phase without additional metadata generation.

The biggest missing layer is not itinerary content. It is archive presentation metadata:

- archive intro
- inspiration taxonomy
- featured reasoning
- cover image mapping
