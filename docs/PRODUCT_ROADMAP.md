# 产品路线图：从 AI 行程草签到 AI 旅行工作台

## 1. 文档目的

这份路线图用来冻结版本边界，避免把多个方向揉进同一轮开发。

当前已完成：

- `v1.0`：一句话输入到完整行程生成闭环
- `v1.1`：点击式积木编辑
- `v1.2`：Mobile Flow First
- `v1.2.5`：真实 AI 冒烟与 mobile 补丁
- `v1.3`：POI / Route / Weather / Route Insight 可信度增强
- `v1.4`：Desktop Workspace UI
- `v1.5`：真实地图基础版

下一站仍是：

- `v1.6`：登录 / 保存 / 我的行程

## 2. 中期方向

产品不是卖票、订酒店或做交易闭环，而是帮自由行新手从“完全不知道怎么安排行程”，走到“有一份可修改、可参考、逐步更可信的旅行计划”。

中期方向保持四个原则：

1. 个性化：方案能反映用户预算、兴趣、节奏和取舍。
2. 省时间：减少开很多网页拼攻略的负担。
3. 可编辑：用户能通过结构化操作持续修改方案。
4. 可执行：地点、路线、天气和节奏越来越接近真实出行。

## 3. 版本总览

| 版本 | 核心主题 | 主要解决的问题 | 核心交付 |
| --- | --- | --- | --- |
| `v1.0` | 先生成一版 | 从零查攻略太慢 | 输入、补充、生成、结果页、复制、导出、重生成 |
| `v1.1` | 点击式积木编辑 | 方案难快速修改 | Day Cabinet、Time Slot、Itinerary Block、Quick Actions |
| `v1.2` | Mobile Flow First | 手机端浏览和编辑体验弱 | `/plan` 三步问卷、`/result` 分页、Pending Changes |
| `v1.2.5` | 真实 AI 冒烟 | Mock AI 不足以验证稳定性 | Qwen 接入、JSON 兼容修复、mobile 补丁 |
| `v1.3` | 可信度增强 | 地点、路线、天气缺少真实依据 | POI 校验、路线估算、节奏提醒、天气派生、route insight |
| `v1.4` | Desktop Workspace UI | 电脑端还不是完整工作台 | 左侧导航 + 中间当前 Day + 右侧 Inspector |
| `v1.5` | 真实地图基础版 | 右侧地图仍是占位 | 高德 JS SDK、真实底图、Day marker、点位列表联动、itinerary block 联动、降级与部署说明 |
| `v1.6` | 登录 / 保存 / 我的行程 | 方案无法沉淀和回看 | 账号系统、保存行程、历史行程、分享链接 |
| `v1.7` | 内容生态 / Inspiration | 缺少灵感与内容沉淀入口 | Inspiration、图片、模板、内容层 |
| `v1.8` | Mobile 工作台同步优化 | mobile 还没有对应工作台形态 | 底部导航、地图全屏页、移动端编辑抽屉 |
| `v1.9 / v2.0+` | 高级编辑工作台 | 生成、验证、编辑、保存仍分散 | 拖拽、地图选点、自动重排、多版本、协作 |

## 4. v1.5：真实地图基础版

### 4.1 一句话定义

把右侧“地图占位 + 点位列表”升级成真实高德地图基础版。

### 4.2 已完成范围

- 高德 JS SDK loader
- 前端 env 读取修复
- 基础地图组件与降级态
- `/result` desktop 真实地图接入
- 当前 Day marker
- marker 与点位列表联动
- itinerary block 与地图联动
- 环境变量说明、手动验收清单、部署准备说明

### 4.3 明确不做

- 地图选点写回
- 拖拽
- 自动重排
- mobile 地图工作台重构
- 登录保存
- 内容生态

### 4.4 已知 Polish

- active marker 视觉样式仍可继续细化
- 这不阻塞 `v1.5` 完成态

## 5. v1.6：登录 / 保存 / 我的行程

### 5.1 一句话定义

让用户的旅行计划能留住、能回看、能分享。

### 5.2 主要方向

- 账号系统
- 保存当前行程
- 历史行程列表
- 分享链接

## 6. 路线图使用规则

1. 一版只解决一个主问题。
2. 每版都必须写清“做什么”和“不做什么”。
3. 没有范围冻结，不进入开发。
4. 没有真实验收结果，不提前升级到下个版本。
