# 产品路线图：从 AI 行程草稿到 AI 旅行工作台

## 1. 文档目的

这份路线图用来冻结版本边界，避免把多个方向揉进同一轮开发。

项目当前已经完成：

- `v1.0`：一句话输入到完整行程生成闭环
- `v1.1`：点击式积木编辑
- `v1.2`：Mobile Flow First、Cabinet Identity、Pending Changes
- `v1.2.5`：真实 Qwen 冒烟与 mobile 补丁
- `v1.3`：POI / Route / Weather / Route Insight 可信度增强

当前下一站：

- `v1.4`：Desktop Workspace UI

## 2. 中期方向

产品不是卖票、订酒店或做交易闭环，而是帮自由行新手从“完全不知道怎么安排行程”，走到“有一份可修改、可参考、逐步更可信的旅行计划”。

中期方向保持四个原则：

1. 个性化：方案能反映用户预算、兴趣、节奏和取舍。
2. 省时间：减少开很多网页拼攻略的负担。
3. 可编辑：用户能通过结构化操作持续修改方案。
4. 可执行：地点、路线、天气和节奏越来越接近真实出行。

## 3. 版本总览

| 版本 | 核心主题 | 主要解决的问题 | 核心交付 |
| --- | --- | --- | --- |
| `v1.0` | 先生成一版 | 从零查攻略太慢 | 输入、补充、生成、结果页、复制、导出、重生成 |
| `v1.1` | 点击式积木编辑 | 方案难快速改 | Day Cabinet、Time Slot、Itinerary Block、Quick Actions |
| `v1.2` | Mobile Flow First | 手机端浏览和编辑体验弱 | `/plan` 三步问卷、`/result` 分页、Pending Changes |
| `v1.2.5` | 真实 AI 冒烟 | Mock AI 不足以验证稳定性 | Qwen 接入、JSON 兼容修复、mobile 补丁 |
| `v1.3` | 可信度增强 | 地点、路线、天气缺少真实依据 | POI 校验、路线估算、节奏提醒、天气派生、轻量 route insight |
| `v1.4` | Desktop Workspace UI | 电脑端还不是完整工作台 | 左侧导航 + 中间当前 Day 计划 + 右侧地图/路线洞察 |
| `v1.5` | 真实地图功能 | 当前只有点位列表和路线摘要，没有地图联动 | 高德地图 SDK、marker、Day 点位联动、积木与 marker 高亮 |
| `v1.6` | 登录 / 保存 / 我的行程 | 方案无法沉淀和回看 | 账号系统、保存行程、历史行程、分享链接 |
| `v1.7` | 收藏 / Explore / Inspiration | 缺少灵感与收藏沉淀入口 | 收藏地点、灵感路线、探索页 |
| `v1.8` | 移动端工作台同步优化 | mobile 还没有对应的工作台形态 | 底部导航、地图全屏页、移动端修改抽屉 |
| `v2.0` | 完整 AI 旅行工作台 | 生成、验证、编辑、保存仍是分散体验 | 拖拽、地图选点加入行程、自动重排、多版本、协作、商业闭环 |

## 4. v1.4：Desktop Workspace UI

### 4.1 一句话定义

把 `v1.3` 的可信度基础接到更成熟的桌面工作台里。

### 4.2 主要方向

- 左侧全局导航
- 中间当前 Day 行程计划
- 右侧地图 / 路线洞察
- 顶部计划操作栏
- 保留 Day Cabinet 和修改篮，不重做主链路

### 4.3 明确不做

- 不做移动端工作台重构
- 不接真实地图 SDK
- 不做登录 / 保存
- 不做完整 Explore / Saved
- 不改 `TripPlan` schema

## 5. v1.5：真实地图功能

### 5.1 一句话定义

把右侧“地图占位 + 点位列表”升级成真实可联动的地图能力。

### 5.2 主要方向

- 高德地图 SDK
- marker 展示
- Day 点位联动
- 当前积木与 marker 高亮同步

### 5.3 暂不承诺

- 不直接做地图选点写回行程
- 不直接做复杂拖拽路线编辑

## 6. v1.6：登录 / 保存 / 我的行程

### 6.1 一句话定义

让用户的旅行计划能留住、能回看、能分享。

### 6.2 主要方向

- 账号系统
- 保存当前行程
- 历史行程列表
- 分享链接

## 7. v1.7：收藏 / Explore / Inspiration

### 7.1 一句话定义

把“规划当前一趟”往“积累以后还会用的地点和灵感”扩一步。

### 7.2 主要方向

- 收藏地点
- Inspiration / 灵感路线
- Explore 页面

## 8. v1.8：移动端工作台同步优化

### 8.1 一句话定义

把桌面端形成的方法，翻译成手机端真正可用的工作台体验。

### 8.2 主要方向

- 底部导航
- 地图全屏页
- 移动端修改抽屉
- mobile 工作台节奏优化

## 9. v2.0：完整 AI 旅行工作台

### 9.1 一句话定义

从“会生成”升级成“能持续编辑、验证、保存和分享”的完整旅行工作台。

### 9.2 主要方向

- 拖拽
- 地图选点加入行程
- 自动重排
- 多版本
- 协作
- 商业闭环

## 10. 路线图使用规则

1. 一版只解决一个主问题。
2. 每版都必须写清“做什么”和“不做什么”。
3. 没有范围冻结，不进入开发。
4. 没有真实验收结果，不提前升级到下一阶段。
