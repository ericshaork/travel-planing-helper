# 产品路线图：从 AI 行程草稿到 AI 旅行工作台

## 1. 文档目的

这份路线图用来冻结版本边界，避免把多个方向揉进同一轮开发。

项目当前已经完成：

- `v1.0`：一句话输入到完整行程生成闭环
- `v1.1`：点击式行程积木编辑
- `v1.2`：Mobile Flow First、Cabinet Identity、Pending Changes
- `v1.2.5`：真实 Qwen 冒烟版与 mobile 表单补丁
- `v1.3`：真实地点与路线可信度增强

当前下一站：

- `v1.4`：Desktop Workspace UI

## 2. 产品中期方向

产品不是卖票、订酒店或做交易闭环，而是帮助自由行用户从“完全不知道怎么安排”，走到“有一份可修改、可参考、逐步更可信的旅行计划”。

中期方向保持四个原则：

1. 个性化：方案能反映用户预算、兴趣、节奏和取舍。
2. 省时间：减少开很多网页拼攻略的负担。
3. 可编辑：用户能通过结构化操作持续修改方案。
4. 可执行：地点、路线、天气和节奏越来越接近真实出行。

## 3. 版本总览

| 版本 | 核心主题 | 主要解决的问题 | 核心交付 |
| --- | --- | --- | --- |
| `v1.0` | 先生成一版 | 从零查攻略太慢 | 输入、补充、生成、结果页、复制、导出、重新生成 |
| `v1.1` | 点击式积木编辑 | 方案难快速改 | Day Cabinet、Time Slot、Itinerary Block、Quick Actions |
| `v1.2` | Mobile Flow First | 手机端浏览和编辑体验弱 | `/plan` 三步问卷、`/result` 分页、Pending Changes |
| `v1.2.5` | 真实 AI 冒烟 | Mock AI 不足以验证真实稳定性 | Qwen 接入、JSON 兼容修复、mobile 补丁 |
| `v1.3` | 真实性增强 | 地点、路线、天气缺少真实依据 | POI 校验、路线估算、节奏提醒、天气派生、轻量 route insight |
| `v1.4` | Desktop Workspace UI | 电脑端工作台形态还不完整 | 左行程右详情 / 地图的成熟桌面工作台 |
| `v1.5` | Mobile Workspace UI | 手机端还没有对应工作台体验 | 底部导航、独立地图页、移动端修改工作台 |
| `v2.0` | 完整旅行工作台 | 生成、验证、编辑、保存仍然分散 | 可编辑、可验证、可保存、可分享、可地图联动的完整工作台 |

## 4. v1.3：真实性增强

### 4.1 一句话定义

让 AI 行程不只是“看起来像攻略”，而是开始有真实地点、坐标、路线距离、通勤时间、天气依据和轻量地图辅助。

### 4.2 已完成范围

- 高德 POI / 地点搜索
- 地点真实性校验
- 地点坐标
- 路线距离与通勤时间
- 单日节奏提醒
- 天气能力增强
- 基础 route insight 展示
- desktop 轻量左右分区雏形
- mobile 独立 route 页

### 4.3 明确不做

- 不做完整 Desktop Workspace UI
- 不做完整 Mobile Workspace UI
- 不做拖拽
- 不接前端地图 SDK
- 不做地图选点写回行程
- 不做保存分享
- 不做登录、数据库、历史版本
- 不改 `TripPlan` schema，除非单独开最小扩展确认

## 5. v1.4：Desktop Workspace UI

### 5.1 一句话定义

把 `v1.3` 的真实性基础接到更成熟的桌面工作台里。

### 5.2 主要方向

- 强化“左行程、右详情 / 地图 / 修改”的桌面布局
- 把“边看边改”做成桌面端主心智
- 保留项目自己的 Day Cabinet / 积木表达

## 6. v1.5：Mobile Workspace UI

### 6.1 一句话定义

把工作台能力翻译成手机端可用的分页、底部导航和抽屉体验。

### 6.2 主要方向

- 底部 Tab
- 独立地图页
- 修改抽屉
- 移动端 Day 卡片重做

## 7. v2.0：完整 AI 旅行工作台

### 7.1 一句话定义

从“会生成”升级成“能持续编辑、验证、保存和分享”的旅行工作台。

### 7.2 主要方向

- 左行程、右地图、详情和修改篮的完整联动
- 更强的地图验证与路线编辑
- 保存、分享、多版本
- 更完整的导入与后续整理能力

## 8. 路线图使用规则

1. 一版只解决一个主问题。
2. 每版都必须写清“做什么”和“不做什么”。
3. 没有范围冻结，不进入开发。
4. 没有真实验收结果，不提前升级到下一阶段。
