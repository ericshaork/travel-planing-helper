# MVP v1.2.5 分阶段实施计划

## 1. 阶段概览

`v1.2.5` 拆成 4 个顺序阶段：

```text
阶段 0：规划文档与范围冻结
  -> 阶段 1：mobile 选项网格修复
  -> 阶段 2：Qwen 真实 AI 接入
  -> 阶段 3：真实 AI 冒烟记录、文档收口、部署准备
```

通用约束：

- 继续复用现有 `TripPlan`、Provider、Prompt、Schema 和 API
- Qwen 通过 `OpenAICompatibleProvider` 接入，不引入平台 SDK
- Mock 继续作为显式可切换的安全模式
- 不做地图、保存分享、拖拽、局部 patch
- 不改 `/api/parse-trip`、`/api/generate-trip` 协议
- 不改 `TripRequest` / `TripPlan` schema

推荐顺序：

1. 本地再跑一轮真实 Qwen 冒烟
2. 校对 Netlify 环境变量
3. 部署 Preview
4. 做 Preview 冒烟
5. 再决定是否切 Production
