# PRD v1.6：登录 / 保存 / 我的行程

## 1. 一句话定义

把项目从“一次性生成一版”推进到“用户可以把自己的行程留住、找回、继续查看”的网站产品。

## 2. 背景

当前 `v1.5` 已经完成：

- `/create -> /plan -> /result` 主生成链路
- 真实 AI、天气、POI、路线与桌面端地图基础能力
- 积木式查看、快捷修改、导出 Markdown、重新生成

但当前状态仍主要依赖 `localStorage`：

- `parsed trip`
- `trip draft`
- `trip request`
- `trip plan`

这意味着：

- 用户换浏览器、换设备、清缓存后，历史计划会丢失
- 用户没有一个稳定的“我的行程”入口回看旧计划
- 用户即使对结果满意，也无法长期保存和继续管理

`v1.6` 要解决的是“账号、保存、找回”，不是重做整个生成器。

## 3. 本轮目标

`v1.6` 的目标只有四个：

1. 用户可以登录和退出。
2. 用户可以把当前生成结果保存到云端。
3. 用户可以在“我的行程”里看到自己的历史计划。
4. 用户可以打开历史计划并回到现有 `/result` 工作台继续查看。

## 4. 范围原则

本轮必须遵守以下原则：

1. 不为数据库接入重构整个项目。
2. 现有 `/create -> /plan -> /result` 链路尽量不动。
3. `localStorage` 继续作为前端临时态。
4. 云端数据库只负责持久化、找回和管理。
5. `/result` 不在 `v1.6` 一开始就改造成完全数据库驱动。
6. 优先做“登录、保存、找回”，不扩成社区、分享或内容平台。

## 5. 用户故事

### 故事 1：未登录用户仍可先生成

- 作为一个首次到访的用户
- 我不登录也能继续用 `/create -> /plan -> /result`
- 这样我不会被账号门槛挡住

### 故事 2：未登录用户点击保存时被引导登录

- 作为一个刚生成出满意方案的用户
- 我点击保存时，如果还没登录，会收到明确引导
- 这样我知道“生成可先体验，保存需要登录”

### 故事 3：登录用户可以保存当前计划

- 作为一个已登录用户
- 我可以把当前 `TripRequest + TripPlan + enrichment` 保存到自己的账户下
- 这样我之后还能回来继续看

### 故事 4：登录用户可以在“我的行程”里看到历史计划

- 作为一个已登录用户
- 我可以看到自己保存过的计划列表
- 这样我不必依赖浏览器本地缓存找回旧内容

### 故事 5：登录用户可以打开历史计划继续查看

- 作为一个已登录用户
- 我可以点开某个历史计划并回到 `/result`
- 这样我仍然在熟悉的工作台里查看地图、路线、天气、导出与修改能力

### 故事 6：登录用户可以更新或删除自己的计划

- 作为一个已登录用户
- 我可以覆盖保存当前版本，也可以删除不再需要的计划
- 这样“我的行程”不会越积越乱

## 6. 页面范围

本轮页面与入口范围：

- `/login`
- `/trips`
- `/result` 的保存入口
- sidebar 中“我的行程”入口

说明：

- `/create`、`/plan`、`/result` 的主生成体验继续保留
- 未登录用户不应因为本轮接入而失去基础生成体验

## 7. 关键体验要求

### 7.1 未登录体验

- 未登录用户可以正常生成计划
- 未登录用户在需要持久化时再被提示登录
- 提示文案要像朋友提醒，不要像拦路墙

### 7.2 保存体验

- 保存按钮位置明确
- 保存成功、失败、未登录三种状态文案清楚
- 不影响导出、重新生成、地图查看、Pending Changes 等既有能力

### 7.3 我的行程体验

- 默认只显示当前登录用户自己的计划
- 列表信息足够帮助用户识别：标题、目的地、日期、创建时间等
- 打开历史计划后仍回到现有工作台，而不是另起一套陌生页面

## 8. 本轮明确不做

- 不做内容社区
- 不做 Explore 正式内容流
- 不做公开分享链接
- 不做多人协作
- 不做付费、订阅或支付
- 不做图片内容库
- 不做收藏攻略体系
- 不做地图选点加入行程
- 不做拖拽编辑
- 不做自动重排
- 不做多版本历史
- 不做复杂权限系统
- 不重构整个 `/result`
- 不修改 `TripPlan` schema
- 不修改 AI / Weather / POI / Route 主链路

## 9. 验收标准

满足以下条件，`v1.6` 才算完成：

1. 未登录用户仍可完成一次完整的生成流程。
2. 登录用户可以保存当前计划到云端。
3. 登录用户可以在 `/trips` 看到自己的历史计划。
4. 登录用户可以从历史计划重新进入 `/result`。
5. 登录用户只能看到、更新、删除自己的计划。
6. 地图、路线、天气、导出、重新生成、Pending Changes 继续可用。
7. 没有把未完成能力伪装成已完成的分享社区或协作系统。

## 10. 与后续版本的关系

- `v1.6` 先解决“账号与计划沉淀”
- `v1.7` 再考虑内容生态、种子内容、灵感卡和半自动内容流水线
- `v1.8` 再优先补 mobile 工作台优化
- 更重的拖拽、地图选点、自动重排、分享协作，继续放在更后版本
