# v1.7 UI Page Audit

## Scope

This audit checks the Phase 3 UI asset binding readiness for:

- Landing: `app/page.tsx`
- Explore home: `app/explore/page.tsx`
- Archive viewer: `app/explore/[id]/page.tsx`
- Workspace: `app/workspace/page.tsx`

This phase does not change UI, styling, component logic, or image files. The goal is to map:

`page -> component -> expected image folders -> current binding status`

## Status Legend

- `Ready`: matching runtime assets already exist and can be bound in Phase 3.1
- `Partial`: some assets exist, but the core visual set is incomplete
- `Missing`: component needs images, but the key runtime assets are not ready
- `Not bound`: assets may exist, but the component currently does not reference them

## Summary

### Pages with the best immediate binding readiness

- Landing
- Workspace empty / feedback states
- Explore inspiration cards for `location` and `season`
- Archive food / POI sections as optional secondary bindings

### Pages with the largest asset gap

- Archive Viewer
- Explore Featured section
- Explore archive list cards
- Workspace save / status expression

### Phase 3.1 implication

The safest first UI asset binding path is:

1. Explore inspiration cards
2. Explore archive list fallback covers
3. Archive cover shell
4. Workspace empty / status states
