# PRD v1.7

## 1. 背景

项目已经在 `v1.6` 完成从生成、地图、天气到登录保存的主功能闭环，产品现在的主要问题不再是“缺少一个核心能力”，而是“现有能力的呈现方式还不够成熟、不够稳定、也不够有产品气质”。

当前 `/result`、`/trips`、`/login`、`/create`、欢迎页等页面已经可用，但在布局空间、信息层级、视觉一致性、状态表达和旅行感氛围上仍有明显优化空间。

因此，`v1.7` 定位调整为一次 UI / UX 大修版本，不新增复杂能力，优先把现有产品修到更像一个成熟、可信、可持续使用的旅行规划产品。

## 2. 目标

`v1.7` 的核心目标：

- 让 `/result` 工作台从“功能堆起来了”升级为“结构更清楚、空间更合理、地图更有价值”
- 让天气、地图、路线、保存状态这些关键模块表达得更像旅行产品，而不是工程调试面板
- 统一图标、状态、卡片和按钮语言，减少页面之间的割裂感
- 强化欢迎页和整体氛围，让产品第一眼更有记忆点
- 在不重写业务链路的前提下，提升整体可理解性、成熟度和使用信心

## 3. 目标用户体验

用户在 `v1.7` 完成后，应能明显感受到：

- 打开 `/result` 后一眼知道先看哪里、下一步做什么
- 地图不再只是一个小预览，而是足够可读的辅助决策区
- 天气模块更关注“要不要带伞、能不能户外、当天适不适合排满”
- 保存、更新、打开历史计划等状态更容易理解，不需要猜当前发生了什么
- landing / create / login / trips 的气质更统一，更像同一个产品
- 页面整体更轻松、清楚、有旅行感，而不是仍停留在 phase 文档展示状态

### 3.1 用户补充体验目标

- 欢迎页先给视觉感受，而不是大量文字说明
- 欢迎页主视觉采用真实景点图 + 水彩手帐背景 + 少量旅行拼贴
- 打开 `/result` 第一眼能同时回答：这是哪趟旅行、当前是哪一天、这一天大概怎么走
- `/result` desktop 应是左右分区的可编辑旅行工作台
- `/result` 默认比例为左侧工作区 `55%`、右侧地图区 `45%`
- 中间通过可拖拽分隔线调整行程区与地图区比例
- 天气表达应更像旅行建议，而不是气象面板
- 页面整体应更接近浅色水彩手帐，不依赖大面积黑色工具按钮

## 4. 范围

### 4.1 v1.7 做

- `/result` 布局与导航修复
- `/result` desktop 双栏工作台
- `/result` 默认左侧 `55%`、右侧 `45%`
- 中间可拖拽分隔线
- 左侧导航遮挡修复
- 默认放大地图
- 天气展示重构为旅行建议表达
- marker / active marker 视觉优化
- 图标系统统一
- 欢迎页海报化
- 欢迎页减少说明文字，强化主视觉和 CTA
- 欢迎页主视觉使用真实景点图，背景加入水彩手帐风，叠加少量旅行拼贴元素
- 欢迎页去除或弱化无实际作用的左侧导航
- 色彩和视觉层级优化
- 建立浅色水彩手帐式视觉强调
- 减少大面积黑色 CTA
- 保存 / 打开 / 更新 / 错误 / 空状态表达优化
- `/create /plan /result /trips /login` 页面一致性整理
- 清理 `/create` 无实际功能入口
- 检查并修复 itinerary block “查看详情”无响应
- mobile 回归检查

### 4.2 v1.7 不做

- 地图选点
- 拖拽编辑
- 自动重排
- 分享
- 协作
- 内容生态
- 灵感卡内容系统
- 路线模板能力
- 数据库结构变化
- AI 主链路变化
- 登录系统大改
- 支付
- 多版本历史
- 不深度重构移动端，`v1.7` 电脑端优先
- 不做复杂专项天气推理
- 不大规模引入复杂插画系统
- 不做满屏贴纸墙
- 不把所有按钮都改成水彩样式

## 7. 非目标

`v1.7` 明确不是以下方向：

- 不是功能大版本
- 不是内容生态版本
- 不是移动端深重构版本
- 不是登录体系重构版本
- 不是地图交互重构版本
- 不是 AI 规划逻辑重构版本

## 8. 成功标准

- `/result` 的左侧导航不再遮挡主要内容
- 欢迎页不再像文字广告或阶段说明页
- 欢迎页第一眼有真实旅行主视觉，同时具备水彩手帐气质
- desktop 工作台空间利用明显改善，地图存在感提升
- `/result` 默认地图面积明显增大
- `/result` 默认左侧工作区 `55%`、右侧地图区 `45%`
- `/result` 能通过拖拽调整行程区与地图区比例
- 天气模块能优先表达旅行决策信息
- marker / 图标 / 状态 / 卡片语言更统一
- `/create` 不再展示明显过期或无实际功能入口
- itinerary block 详情入口可正常响应，或不可用时有明确状态
- 主 CTA 不再依赖大面积黑色块
- `/trips` 和 `/login` 不再显著保留旧 phase 文案语境
- 全流程 UI 回归后，不影响现有生成、保存、打开、更新、删除能力
- desktop 观感明显提升，mobile 不严重破版

## 9. 验收标准

- `/create -> /plan -> /result` 主链路仍可完整跑通
- 登录后可以保存当前计划
- `/trips` 可以打开、更新、删除历史计划
- 地图、天气、路线洞察主链路不被 UI 改造破坏
- 关键状态有明确反馈：保存中、已保存、更新中、已更新、错误、空状态
- 页面之间的按钮、chip、icon、卡片和语气明显更统一
- lint / build / typecheck / test 通过
