# v1.7 Phase 3.5.2 Workspace Map / Route Visual Integration

## Scope

This phase continues the Workspace visual integration without changing:

- TripPlan data structure
- AI generation flow
- Explore and Archive flow
- Supabase or API contracts
- map SDK logic
- route calculation logic

The goal is to make the right side of Workspace read like a travel planning board instead of a stack of unrelated tools.

## Map Visual Rules

- `MapPanel` now acts as the outer shell for the full route area.
- The shell adds:
  - paper background noise
  - bookmark accent
  - route header
  - quick chips for stops, distance, and duration
- The map itself stays unchanged and still uses the existing AMap logic.
- `InspectorMapPreview` frames the map as a route board rather than a raw preview widget.

## Route Display Rules

- `WorkspaceInspector` now introduces the route rail as a travel archive / route board.
- `InspectorPointList` is reshaped into a simple stop timeline:
  - numbered stops
  - vertical sequence line
  - slot context
  - map status
- `InspectorPointDetailCard` is now positioned as a stop note.
- `InspectorRouteLegs` reads as travel segments rather than technical legs.
- `InspectorRouteStats` keeps compact planning stats for mapped stops, pending stops, distance, and time.

## Weather Rules

- `WeatherAlertCard` now uses `ui/weather/*` assets through a resolver.
- Weather is framed as travel impact first, forecast second.
- The top area emphasizes:
  - weather state
  - main overview
  - travel impact summary
- Forecast rows and alerts remain intact, but now sit inside a lighter travel-note presentation.

## Transport Rules

- `TransportAdvice` now uses `ui/transport/*` assets through a resolver.
- The section starts with:
  - a transport scene background
  - a primary mode visual
  - the existing summary text
- Each transport option keeps the same content logic, but now reads more like a travel planning card than a plain option list.

## Right Rail Rhythm

The right side now follows a clearer order:

1. map shell
2. route note
3. map preview
4. stop note
5. route stats
6. stop timeline
7. route segments
8. pace / weather notes

This makes the inspector feel like one planning surface with sections, instead of multiple disconnected cards.

## Desktop / Mobile Notes

- Desktop gets the main benefit because the map / route rail is visually grouped.
- Mobile continues using the same weather and transport cards, so image ratio and content density improve there too.
- This phase does not redesign mobile workspace structure.

## Remaining Gaps

- Route cards still rely on text-heavy data from the current enrichment output.
- The editor and route rail are more aligned now, but their section rhythm can still be tightened further.
- Weather and transport content are visually stronger, but budget / hotel / attraction sections still keep an older treatment.
