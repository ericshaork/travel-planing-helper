## 7. 抽屉阅读模板的图片需求

基于当前 200 篇真实数据与 `ArchiveReaderViewModel` 目标，第一版抽屉的图片需求非常清楚。

### 7.1 每篇文章是否必须有 hero cover

第一版如果要“像档案阅读”，必须有。

原因：

- 当前 200 篇没有任何 `coverImageUrl`
- 只有文字和城市卡图，会让抽屉看起来像“文本列表 + 纸张壳子”
- 至少核心展示批次的文章必须有档案级 hero cover

### 7.2 是否需要 `galleryImages`

第一版不需要强制。

原因：

- 原始 JSON `0/200` 都没有 gallery
- 现在硬要求会直接卡住
- 可以先允许为空，或者只给少数重点档案补 1-2 张次级图

### 7.3 `food` 是否需要每条配图

第一版不需要每条配图。

建议：

- 先用 shared food fallback
- 一组城市/主题共用一张 food 图
- 不必对每条菜名、每个小吃点单独生成

### 7.4 `places/pois` 是否需要每条配图

第一版也不需要每条配图。

建议：

- 先用 city / theme / location fallback
- 重点城市后续再逐步补 대표 POI 图

### 7.5 哪些可以先用 fallback

可以先用 fallback：

- `galleryImages`
- `food` 图
- `poi` 图
- `recommendedFor` 展示配图
- 节奏类/同行人类主题图

### 7.6 哪些必须优先补

必须优先补：

1. 档案级 hero cover
2. 高量级主题 fallback
3. 少量正文共用插图

### 7.7 如果只做 v1.8 可展示版本，最少需要补多少张图

务实建议：

- `24-36` 张档案 hero cover
- `8-12` 个主题 fallback
- `6-10` 张正文共用插图

合计建议补图规模：

- 约 `38-58` 张

这已经足够把 Explore 档案抽屉做成“可以拿给朋友看”的版本。

### 7.8 如果要把 200 篇都做到比较完整，大概需要多少张图

分两档看：

#### 最小完整档

- `200` 张档案 hero cover
- `8-12` 个主题 fallback
- `20` 个城市 fallback archive cover
- `20-40` 张正文共用插图

大约：

- `248-272` 张左右

#### 更像 editorial 阅读产品的完整档

- `200` 张档案 hero cover
- `100-200` 张正文插图
- `20-40` 张 food/poi 共用图
- `8-12` 个主题 fallback

大约：

- `328-452` 张左右

结论：

- 第一版完全没必要一口气补到 200 全量完整
- 最合理的是先做“首批封面 + 高量级主题 fallback”

## 11. 风险与待确认问题

1. 当前 200 篇原始 JSON 全都没有 `coverImageUrl`，这不是少量漏图，而是源数据层完全没图。
2. 当前 200 篇原始 JSON 全都没有 `rawContent`，说明抽屉的“旅行故事 / intro / suitable for”必须由 normalize 与推导层生成。
3. 当前 200 篇原始 JSON 全都没有 `theme`，需要单独建立 tags -> theme bucket 的映射。
4. `slug` 有 7 组重复，导入时必须去重，否则后续 `/explore/[slug]` 与 drawer key 会冲突。
5. 当前 `city.name` 是英文城市名，正式中文产品展示不能直接拿来当前台城市文案。
6. 虽然正文图片字段全缺，但 `summary + daily_itinerary + pois + food + tags` 已足够支撑第一版抽屉文字内容。
7. 仓库里仍有国外城市图片，但当前这 200 篇实读数据没有国外内容，不应混入本轮中国版补图优先级。

## 12. 务实结论

这次重新实读 200 篇之后，结论比旧推断更明确：

- 当前真正缺的不是正文文本，而是图片契约
- 这 200 篇的文字骨架其实很整齐
- 但图片相关字段几乎是 `0/200`

所以 v1.8 第一版最合理的补图策略不是“做全”，而是：

1. 先补 `24-36` 张首批档案 hero cover
2. 先补 `8-12` 个高频主题 fallback
3. 先补 `6-10` 张正文共用插图

只要先把这一批补起来，Explore 档案抽屉就能从“有文字但视觉单薄”升级到“可展示、可阅读、能拿给朋友看”的版本。
