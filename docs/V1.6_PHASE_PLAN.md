# V1.6 Phase Plan

## 总目标

在不重构现有旅行生成主链路的前提下，补齐：

- 登录
- 云端保存
- 我的行程
- 历史计划找回
- 已保存计划更新 / 删除

## 阶段总览

### Phase 0

- 文档冻结
- 明确 Supabase 方案

状态：已完成

### Phase 1

- 安装 Supabase SDK
- 增加 env helper
- 增加 browser / server helper

状态：已完成

### Phase 2

- `profiles` / `trip_plans`
- migration
- RLS
- trigger

状态：已完成

### Phase 3

- `/login`
- email magic link
- Header 登录态
- 退出登录

状态：已完成

### Phase 4

- `/result` 保存当前方案
- `POST /api/trips`

状态：已完成

### Phase 5

- `/trips`
- `GET /api/trips`
- Header / sidebar 我的行程入口

状态：已完成

### Phase 6

- `GET /api/trips/[tripId]`
- 从 `/trips` 打开历史计划
- 恢复到现有 localStorage
- 跳回 `/result`

状态：已完成

### Phase 7

- `savedTripId` metadata
- “保存新计划”与“更新已保存计划”自动分流
- `PATCH /api/trips/[tripId]`
- `DELETE /api/trips/[tripId]`
- `/trips` 删除交互
- `/result` 保存状态收口

状态：已完成

实现边界：

- 不新建数据库驱动版 `/result`
- 不改 Trip schema
- 不改 AI / Weather / Map / Route 逻辑
- 不做分享
- 不做多版本历史

### Phase 8

- 总验收
- 文档收口
- 环境变量说明收口
- roadmap 更新
- 发布准备

状态：已完成

## v1.6 最终验收结果

- 登录 / 登出已可用
- `/result` 可保存当前计划到云端
- `/trips` 可查看自己的历史计划列表
- 历史计划可回到现有 `/result`
- 已保存计划可更新
- 已保存计划可删除
- 仍然只允许操作当前登录用户自己的 trip
- `lint / build / typecheck / test` 通过

## 当前仍然不做

- `/trips/[id]`
- 分享
- 多版本历史
- 纯数据库驱动工作台
- Trip schema 变更
- v1.7 UI 大修
