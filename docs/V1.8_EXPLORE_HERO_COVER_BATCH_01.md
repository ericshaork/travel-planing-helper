# V1.8 Explore Hero Cover Batch 01

## 一、批次目标

本批次只规划 `30` 张档案级 `Hero Cover`，用于：

- Explore 档案卡片主封面
- Archive Drawer 顶部主视觉
- 未来文章详情页封面
- 分享预览图候选

这批图**不是** city-card，也**不是**正文插图。  
city-card 继续承担城市维度 fallback；本批次的目标是把“文章级主视觉”先补齐一轮，让首批最值得展示的档案拥有自己的封面气质。

## 二、选择规则

本批次从以下数据源交叉筛选：

- `data/explore/normalized/explore_archives_v1.json`
- `data/explore/normalized/explore_archive_image_manifest.json`

筛选原则：

1. 先覆盖 20 个核心城市，每城至少 1 篇。
2. 优先使用 manifest 中已标记 `P0` 的档案。
3. 当某些城市没有合适的 `P0`，或首批需要补齐 `family / heritage / night / citywalk / seaside / jiangnan` 等主题时，补少量 `P1` 高质量候选。
4. 优先选择 `title / summary / dailyItinerary / places / food` 都较完整、画面感强、城市识别度高的文章。
5. 同一城市最多优先拿 `1-2` 篇，只有北京、上海、成都、重庆、敦煌、西安、厦门、青岛、杭州这类视觉辨识度或主题扩展价值特别强的城市，会进入第二篇。

## 四、生成提示词规范

本批次 30 张 Hero Cover 建议统一遵循同一套风格基线：

- 横版城市旅行档案卡
- 左侧保留纸张留白和档案感呼吸区
- 右侧为城市主景水彩 / 半写实主视觉
- 暖米色纸张质感
- 少量邮戳、花枝、纸胶带、小票、地标或食物装饰
- 复古旅行杂志感，不要 SaaS，不要科技感

统一负面约束：

- 不要生成英文大段文字
- 不要把文字作为主体，后续标题由代码叠加
- 不要生成错误城市地标
- 不要生成国外城市元素
- 不要大面积黑色 UI
- 不要过度赛博、未来感、霓虹科技感
