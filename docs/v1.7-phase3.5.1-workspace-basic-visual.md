# v1.7 Phase 3.5.1 Workspace Basic Visual Integration

## Scope

This phase applies a first-pass visual integration to Workspace without changing:

- TripPlan data structure
- AI generation flow
- Explore and Archive flow
- Supabase or API contracts
- Workspace editing logic

The goal is to make Workspace feel like the editing counterpart to Explore and Archive: lighter than an archive viewer, but still clearly part of the same travel product language.

## Header Rules

- The top bar is treated as the travel editing entry, not as a generic dashboard header.
- The first row shows product context: `TRAVEL WORKSPACE`, destination, and route insight state.
- The main title remains the trip title.
- The summary stays readable and secondary.
- Action buttons remain in the top area, but the save status now appears as a dedicated visual status block.
- Fact cards focus on:
  - destination
  - days
  - dates
  - budget
  - current focus day

## Sidebar Rules

- Sidebar keeps the existing navigation logic and item structure.
- Active state is expressed with:
  - a left indicator bar
  - a lighter journal-like card background
  - a soft hover accent image inside the icon container
- Hover and active states now feel less like admin navigation and more like a travel toolrail.
- Unavailable sections still render as `Soon` and keep placeholder behavior.

## Save Status Rules

- Save state is now expressed with both text and a visual accent image.
- States are normalized to:
  - `Ready to save`
  - `Login to save`
  - `Saving...`
  - `Saved`
  - `Saved draft`
  - `Need attention`
- The save button copy is now readable English instead of legacy garbled text.
- Save logic and persistence flow are unchanged.

## Visual Language Alignment

Archive and Workspace now share:

- soft paper background accents
- bookmark / label style decorative cues
- warm layered panels
- concise editorial-style kicker labels

The difference is intentional:

- Archive = reading a travel record
- Workspace = editing a live travel plan

## Current Limits

- Mobile workspace copy is still mostly legacy text and was not part of this phase.
- Map and route panels still keep their earlier functional visual treatment.
- Workspace content cards are not yet unified with the new top-level language.
- No animation or larger layout restructuring was introduced in this phase.

## Next Phase Suggestion

Phase `3.5.2` should focus on map / route visual integration:

- MapPanel shell
- route inspector emphasis
- weather / transport card assets
