# v1.7 Phase 3.6.0 - User Flow Assembly

## 背景

当前项目已经具备以下能力：

- Explore 数据链路：JSON -> Importer -> Supabase -> API -> Explore 页面
- Explore 首页骨架：City Search / Inspiration / Featured / Feed
- Travel Archive Viewer：Desktop Drawer + Mobile Full Page
- Create My Version：Explore / Archive -> TripPlanDraft -> Create
- AI Planning 主流程：Create -> Plan -> Generate
- Workspace：三栏编辑空间

Phase 3.6.0 的目标不是继续打磨单个组件，而是把这些模块串成一个用户第一次访问就能走通的完整产品流程。

## 本阶段发现的入口问题

### 1. `/` 不是 Landing，而是旧的 Workspace landing

之前首页直接渲染 `WorkspaceLanding`，这会带来两个问题：

- 新用户第一次进入站点时，先看到的是偏 workspace 的说明，不是产品入口；
- 主路径变成“先看工作台，再找怎么开始”，不符合 Landing -> Explore / Create 的产品顺序。

### 2. `/plan` 生成完成后仍然跳到 `/result`

当前产品定位里，`/workspace` 才是正式的编辑空间入口。  
如果生成完成后仍回到 `/result`，用户会继续停留在兼容路由，而不是进入新的主编辑流。

### 3. `/workspace` 无计划时的空状态不够明确

之前无行程时，页面仍更偏向旧的 “回 `/plan` / 回 `/create`” 语义。  
对第一次进来的用户来说，需要更明确地提示：

- 这个 Workspace 现在还是空的；
- 可以去 Create 直接开始；
- 也可以先去 Explore 找灵感。

## 本阶段实施内容

## 1. Landing 入口恢复为真正首页

将 `/` 从 `WorkspaceLanding` 切换为新的 `LandingPage`。

Landing 当前职责：

- 解释产品现在的主流程；
- 提供两个明确入口：
  - `Start exploring` -> `/explore`
  - `Start planning` -> `/create`
- 提供一个次级入口：
  - `Open workspace` -> `/workspace`

这让首页重新承担“第一次访问入口”的职责，而不是直接承担编辑空间说明页。

## 2. AI 生成完成后进入 `/workspace`

在 `app/plan/page.tsx` 中，将成功生成后的跳转从：

- `/result`

调整为：

- `/workspace`

这样主用户链路就变成：

Create -> Plan -> AI Generate -> Workspace

`/result` 继续保留为兼容入口，不作为主路径。

## 3. Workspace 空状态改为产品化入口

在 `WorkspaceRoutePage` 中补充了区分：

- `mode="workspace"` 时：
  - 明确显示 `WORKSPACE EMPTY`
  - 主操作：`Create your first trip`
  - 次操作：`Browse Explore`

- `mode="result"` 时：
  - 继续保留结果缺失语义
  - 主操作：`Back to plan`
  - 次操作：`Start over`

这样 `/workspace` 在没有本地行程时也不会变成死路。

## 当前页面职责

### `/`

Landing 页面。

职责：

- 产品介绍
- 两个主入口
- 解释当前完整流程

### `/explore`

Travel Discovery Hall。

职责：

- 城市搜索
- 灵感探索
- 精选档案
- 内容流入口

### `/explore/[id]`

Mobile Archive Page 容器。  
Desktop 仍由 Explore 首页中的 drawer 打开 Archive。

职责：

- Archive 阅读
- Create My Version

### `/create`

Create 入口页。

职责：

- AI-assisted 入口
- 接收 Explore draft
- 进入 `/plan`

### `/plan`

最终确认与生成页。

职责：

- 补全需求
- 生成 TripPlan
- 成功后进入 `/workspace`

### `/workspace`

正式的旅行编辑空间。

职责：

- 打开当前 TripPlan
- 继续编辑
- 查看地图、路线、保存状态

### `/result`

兼容入口。

职责：

- 保持旧链接可用
- 不再作为主用户流的目标页

## 当前主用户流程

### 新用户路径

Landing  
-> Explore 或 Create  
-> Archive / Inspiration / City Search  
-> Create My Version 或直接创建  
-> Plan  
-> AI Generate  
-> Workspace

### Explore 路径

Landing  
-> `/explore`  
-> 选择 Inspiration / Featured / Feed 卡片  
-> Desktop 打开 Archive Drawer，Mobile 打开 Archive Page  
-> `Create My Version`  
-> `/create`  
-> `/plan`  
-> `/workspace`

### 直接创建路径

Landing  
-> `/create`  
-> `/plan`  
-> `/workspace`

## 无状态处理

### 无 TripPlan 进入 `/workspace`

展示 Empty Workspace，引导用户：

- 直接创建第一条旅行
- 或先浏览 Explore

### 无草稿进入 `/plan`

继续保留已有兜底：

- 回 `/create`
- 或回首页重新开始

### 无 Explore 数据

本阶段未改动数据层。  
继续复用现有 Explore fallback 与 API 返回处理。

## 后续建议

### 1. Landing 文案与视觉在后续 UI 阶段继续收口

本阶段 Landing 重点是恢复正确入口职责，后续再做视觉精修。

### 2. `/result` 后续可以逐步降级为兼容页

当前仍建议保留，避免旧链路或书签失效。  
后续可以考虑：

- 在 `/result` 增加提示，引导用户去 `/workspace`
- 或长期保留为别名入口

### 3. 再做一轮端到端人工验收

建议重点验证：

- `/` -> `/explore`
- `/` -> `/create`
- `/explore` -> Archive Drawer / Archive Page
- `Create My Version` -> `/create`
- `/plan` 生成后 -> `/workspace`
- 空状态进入 `/workspace`
