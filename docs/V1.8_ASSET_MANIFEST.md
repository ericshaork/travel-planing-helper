# Wanderly v1.8 Asset Manifest

## 1. 文档用途

本文档用于盘点 Wanderly 当前仓库中的图片资产现状，明确：

- 现有图片在哪里；
- 哪些已被代码明确引用；
- 哪些属于可能的动态引用；
- 哪些更像 source-only 或候选素材；
- 哪些不应进入 v1.8 正式页面；
- 后续应如何处理而不破坏现有代码引用。

本文档只输出判断和建议，不执行任何图片处理动作。

## 2. 盘点范围与方法

本次盘点只读检查了以下内容：

- `public/images/` 全目录；
- `public` 下是否存在 `images/` 之外的图片；
- `public/images/*/README.md` 与 `public/images/IMAGE_INDEX.md`；
- Landing、Explore、Workspace、Header 等当前主要组件中的图片引用；
- `lib/explore/image-resolver.ts`；
- `lib/trip/workspace-asset-resolver.ts`；
- `lib/trip/workspace-visuals.ts`；
- `lib/explore/mock-archives.ts`；
- 全项目中对 `/images/` 的明确字符串引用；
- CSS 中是否存在 `background-image: url(...)` 或其他图片背景引用。

判断口径：

- A. 明确被引用：代码中能找到明确路径；
- B. 可能动态引用：通过 resolver、映射规则或数据字段拼接；
- C. 当前未发现引用：代码与解析逻辑中暂未发现。

注意：

- 旧 README 明显已过时，只能作为历史参考；
- “当前未发现引用”不等于“可删除”；
- `_source/` 目录按 source-only 处理，不直接视为运行时资产。

## 8. 透明背景情况

### 8.1 当前结论

从文件元数据角度看：

- 当前 PNG 全部带 alpha 通道；
- 未发现批量“应该透明但没有透明”的 PNG。

### 8.2 风险解释

这不代表所有图片都适合当前页面，原因在于：

- 带 alpha 通道不等于裁切合理；
- 带 alpha 通道不等于素材本身适合桌面端正式页面；
- 很多图即使有透明通道，也可能仍属于 source-only、国际候选或风格过旧资产。

因此透明背景的主要结论是：

- 当前不是“缺 alpha”问题；
- 当前是“是否该进入正式页面”“是否需要重裁切/重生成/归档”的问题。

## 16. Settings 资产

## 16.1 当前实际情况

- `icons/outline/`：`REVIEW`
- `ui/state/`：`KEEP`
- Settings 大图、背景图、装饰图：当前应视为 `MISSING but probably unnecessary`

## 20. 安全处理顺序

后续建议严格按以下顺序处理，而不是先删图：

1. 确认代码明确引用；
2. 确认 resolver 与动态映射；
3. 用户人工视觉确认；
4. 冻结 KEEP / ARCHIVE / SOURCE_ONLY 边界；
5. 再处理 CROP / TRANSPARENT / REPLACE / REGENERATE / COMPRESS；
6. 正式接入页面；
7. 截图验收；
8. 最后才考虑归档或清理未引用候选。

## 21. 不得自动删除的内容

本次盘点明确禁止自动删除：

- `_source/` 全部内容；
- 当前运行时目录中尚未确认的候选图；
- 与国际内容相关的现有素材；
- 与 `workspace/` 目录相关的未接线成套资源；
- 与 `legacy-duplicates/` 对应的任何文件。

原因是：

- 其中很多仍可能承担未来页面精修的人工筛选来源；
- 其中很多已与历史整理结构绑定；
- 当前删除风险远大于保留风险。
