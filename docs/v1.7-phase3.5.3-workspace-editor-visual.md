# v1.7 Phase 3.5.3 Workspace TripEditor Visual Integration

## Scope

This phase upgrades the middle Workspace editor from a functional itinerary surface into a travel journal editing space, without changing:

- TripPlan data structure
- AI generation logic
- Explore / Archive flow
- map logic
- Supabase / API contracts

## TripEditor Visual Rules

- The middle column should read as a travel editing journal, not a backend form.
- Summary, day chapters, current day panel, edit desk, and export all use the same paper / bookmark / label family now.
- Editing affordances stay visible, but they are embedded into journal-like content blocks.

## Day Structure Rules

- `WorkspaceDayTabs` behaves like chapter navigation.
- Each day card emphasizes:
  - day number
  - theme
  - date
  - number of journal blocks
- `WorkspaceDayPanel` frames the chosen day as the active chapter.
- `DayCabinet` keeps Morning / Afternoon / Evening, but presents them as travel journal sections rather than plain grouped rows.

## Modification Entry Rules

- `ModificationPanel` now acts as an AI edit desk.
- The flow is:
  - pending edits
  - quick direction
  - rewrite request
  - regenerate
- `PendingChangesPanel` and `ModificationQuickActions` are intentionally lightweight, so they support the journal instead of overtaking it.

## Export Rules

- Export is treated as a finishing action rather than a generic utility.
- Copy and download are still the same logic, but the section now reads as:
  - finish the journal
  - take the trip away
  - keep a saved workspace version

## Desktop / Mobile Notes

- Desktop gets the clearest benefit because the middle column now stands as its own editorial surface between sidebar and route rail.
- Mobile inherits the cleaner card treatments and copy hierarchy through shared components.
- This phase does not redesign the mobile workflow itself.

## Remaining Gaps

- Some helper components in other trip reference areas still keep older text or older card rhythm.
- Budget / hotel / attraction reference cards are still less aligned than the main journal editor.
- There is still room to better differentiate “reading archive” versus “editing journal” through finer spacing and motion in later phases.

## Next Suggestion
