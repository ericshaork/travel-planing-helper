# 竞品洞察：从参考灵感到版本取舍

## 1. 竞品洞察总览

这份文档不把竞品拆成一条条流水账，而是按能力归类，回答两个问题：

1. 用户真正认可的是什么；
2. 这些能力应该进入 v1.2、v1.3、v1.4、2.0 的哪一层。

### 1.1 工作台布局

- Wanderlog 给出的核心启发是：左侧工作台 + 右侧地图的双栏结构很容易建立“边看边改”的工作心智；
- Mindtrip 进一步说明，左侧不只是列表，而应是可选中、可高亮、可联动的计划面板；
- 对本项目的翻译不是直接照搬“列表 + 地图”，而是未来形成“左侧行程柜 + 积木块，右侧地图”的工作台。

### 1.2 地图能力

- Wanderlog、Roadtrippers、Mindtrip 都证明：地图不是装饰，而是用户判断路线是否合理、地点是否顺路的核心界面；
- 用户明确希望未来支持拖动、缩放、点位详情、路线展示，以及从地图选点加入左侧计划；
- 这类能力对产品长期价值很强，但会显著抬高数据、Provider、交互和状态复杂度，因此不应提前塞进 v1.2。

### 1.3 输入方式

- Roadtrippers 的分层选择说明：用户并不总想从一大段自然语言开始，结构化入口也有价值；
- TripIt 的“已有计划整理”提醒我们，未来不只有“从零生成”，还要支持“我已经有一部分计划，请帮我补全”；
- 因此，本项目未来应同时保留：
  - 一句话输入；
  - 分步补信息；
  - 已有计划导入 / 补全入口。

### 1.4 编辑方式

- Wanderlog 的工作台思路说明，左侧应以卡片、块、结构化编辑为主；
- 用户同时强调：打字修改不能消失，它仍然是低成本表达复杂约束的辅助方式；
- Mindtrip 的卡片与地图联动则说明，未来编辑不能只停留在“改文案”，而要形成“卡片、地图、路线”三方联动。

### 1.5 已有计划导入

- TripIt 的价值不在“帮你想去哪”，而在“把已有行程整理完整”；
- 对本项目的启发是：未来应支持用户带着机票、酒店、部分景点、已有安排进入，再由系统补全完整方案；
- 这更接近旅行真实使用场景，也能拉开与“只会从零生成”的产品差异。

### 1.6 图片 / 视频

- Layla 的图片、短视频和路线动效能帮助用户更快判断“这里到底想不想去”；
- 这类内容更适合放在中后期，用于辅助决策和提升沉浸感；
- 当前版本不应为了视觉热闹而提前引入媒体流、视频资源和复杂动效。

### 1.7 商业闭环

- Wanderlog 给用户的预期是：未来酒店、预订、商业入口可以挂在计划旁边；
- 但当前项目尚无真实酒店、机票、门票等商业 API 条件；
- 因此后续最多先做弱入口或弱占位，不做真实下单闭环，更不应在 v1.2 提前绑定商业实现。

### 1.8 Explore 发现页

- Mindtrip 的 Explore 说明，除了“已有需求 → 出方案”，用户未来还会有“先逛、先找灵感、再决定去哪”的场景；
- 这是一条完整的新导航支线，不属于当前 v1.2 的生成与编辑主链路；
- 它应进入 2.0 级别工作台规划，而不是在 v1.2 提前做半成品。

## 2. 已纳入 v1.2 的内容

v1.2 只吸收竞品里已经能够直接提升当前主链路、且不要求地图 / 商业 / 持久化支撑的部分：

- Mobile Flow First；
- 柜子 / 抽屉式视觉；
- 积木块编辑；
- Pending Changes 修改篮；
- 打字修改作为辅助方式。

翻译成产品语言就是：

- 学 Wanderlog / Mindtrip 的“工作台心智”，但先只落到柜子、积木和修改篮；
- 学移动端分层与单屏任务，而不是照搬竞品的完整地图工作台；
- 保留打字修改，不把产品做成只有点击没有表达自由度的纯按钮工具。

## 3. v1.3 建议加入

v1.3 适合吸收的是“真实性基础层”，而不是完整工作台外观：

- 真实 AI 接入；
- 高德 POI；
- 路线距离 / 时间；
- 天气；
- 自驾 / 公交 / 步行等出行方式；
- 地图基础展示；
- 地点卡和地图点位的基础联动。

对应竞品来源：

- Wanderlog / Mindtrip：地图是计划验证界面，不只是配图；
- Roadtrippers：路线地图和出行方式值得吸收，尤其后续加入“自驾”；
- Layla：路线可视化值得参考，但先不做重动效版本。

## 4. v1.4 建议加入

v1.4 更适合吸收“留存、补全、分享”这层能力：

- 保存方案；
- 分享链接；
- 历史版本；
- 已有计划导入入口；
- 用户已有机票 / 酒店 / 想去地点时，可帮其补全方案；
- 商业按钮只做弱占位，不做真实预订闭环。

对应竞品来源：

- TripIt：已有计划整理与补全；
- Wanderlog：未来可挂商业入口，但先只做弱入口；
- 这一步的重点不是让产品更花，而是让计划能留住、能回看、能接着改。

## 5. 2.0 建议加入

2.0 才适合正式吸收“工作台级”竞品能力：

- 左侧行程柜 + 右侧地图工作台；
- 地图拖动、缩放、点位详情；
- 地图选点加入左侧工作台；
- Explore 发现页；
- Saved / Inspiration / Notes / Trips 等工作台导航；
- 地图与卡片双向高亮；
- 逐步形成类 Wanderlog / Mindtrip 的工作台效果，但保留本项目的柜子 + 修改篮特色。

补充说明：

- Layla 的路线连线动效、图片 / 短视频辅助判断，也更适合作为 2.0 之后的增强项；
- 2.0 的目标不是做“另一个 Wanderlog”，而是做“更适合 AI 生成 + 柜子编辑 + 修改篮”的旅行计划工作台。

## 6. 暂不做

以下能力明确不进入 v1.2：

- 不在 v1.2 做地图；
- 不在 v1.2 做图片 / 短视频；
- 不在 v1.2 做酒店 / 机票 / 门票商业 API；
- 不在 v1.2 做真实保存分享；
- 不在 v1.2 做地图选点加入行程；
- 不在 v1.2 做 Explore 发现页。

一句话边界：

v1.2 吸收的是“移动端流程、柜子识别、积木编辑、修改篮”；
地图、商业、保存、灵感发现与媒体展示全部后置。
