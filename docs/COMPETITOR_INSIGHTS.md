# 竞品洞察：从参考灵感到版本取舍

## 1. 这份文档回答什么

这份文档不做竞品百科，而是回答两件事：

1. 哪些能力真的值得吸收。
2. 这些能力应该进 `v1.3`、`v1.4`、`v1.5` 还是 `v2.0`。

## 2. 主要竞品带来的启发

### 2.1 Wanderlog

- 强项是桌面端“左计划、右地图”的工作台心智；
- 说明地图不是装饰，而是验证路线合理性的核心界面；
- 也说明桌面工作台一旦做，就会把交互复杂度显著拉高。

### 2.2 Mindtrip

- 强项是卡片、地图和详情之间的联动；
- 提醒我们未来工作台不能只停留在纯文本阅读；
- 但这种联动更适合后续完整工作台，而不是当前基础版。

### 2.3 Roadtrippers

- 强项是路线、出行方式、地图和点位关系；
- 对 `v1.3` 最有启发，因为它对应的正是路线可信度和地图基础验证。

### 2.4 TripIt

- 强项是“整理已有行程”；
- 提醒我们后续不仅要支持从零生成，也要支持带着已有计划进来补全；
- 这类能力更适合后续留存与工作台阶段。

### 2.5 Layla

- 强项是路线可视化、图片和短视频氛围；
- 对长期沉浸感有帮助；
- 但不适合当前先行投入。

## 3. 各版本吸收原则

### 3.1 v1.3

`v1.3` 只吸收真实性基础，不吸收完整工作台外观。

本版适合吸收：

- 真实地点校验；
- 路线距离和时长；
- 基础地图验证；
- 节奏提醒；
- 天气真实性增强。

本版不吸收：

- 竞品完整桌面 UI；
- 移动端完整工作台 UI；
- 复杂卡片地图联动；
- 拖拽和工作台级编辑。

补充冻结结论：

- 竞品桌面 UI 进入 `v1.4`；
- 竞品移动端工作台 UI 进入 `v1.5`；
- `v1.3` 只做真实性基础层。

### 3.2 v1.4

`v1.4` 适合吸收桌面工作台 UI。

重点：

- 左侧行程；
- 右侧地图、详情、修改篮；
- 更强的“边看边改”体验；
- 但要保留本项目自己的 Day Cabinet / 积木表达。

### 3.3 v1.5

`v1.5` 适合吸收移动端工作台 UI。

重点：

- 底部导航；
- 独立地图页；
- 修改抽屉；
- 手机端工作台节奏。

### 3.4 v2.0

`v2.0` 才适合整合完整旅行工作台能力。

重点：

- 地图与卡片联动；
- 更完整的编辑能力；
- 保存与分享；
- 多版本与导入；
- 更成熟的工作台结构。

## 4. 当前明确不提前做

- 不在 `v1.3` 提前做完整桌面工作台；
- 不在 `v1.3` 提前做移动端工作台重构；
- 不在 `v1.3` 提前做拖拽；
- 不在 `v1.3` 提前做地图选点写回行程；
- 不在 `v1.3` 提前做商业预订闭环；
- 不在 `v1.3` 提前做图片/视频内容流。

## 5. 结论

竞品给我们的最大启发不是“照着做一个像它们的界面”，而是把能力按顺序拆开：

1. 先补真实性基础；
2. 再补桌面工作台；
3. 再补移动端工作台；
4. 最后整合成自己的完整旅行工作台。
