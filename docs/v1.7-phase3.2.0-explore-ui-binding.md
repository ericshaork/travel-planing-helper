# v1.7 Phase 3.2.0 Explore UI Binding

## Scope

This phase formalizes the first production-style image binding on the Explore home page.

It does not include:

- layout redesign
- animation
- Archive visual development
- Workspace visual development
- image generation

## Bound Components

### `InspirationDeck`

Files:

- `components/explore/InspirationDeck.tsx`
- `components/explore/InspirationCard.tsx`
- `lib/explore/image-resolver.ts`

Binding rule:

- `InspirationDeck` passes only the facet key
- image paths are resolved centrally through `getInspirationCardImageCandidates`
- `InspirationCard` renders the resolved image through `ResolvedImage`

## Shared Runtime Rules

Component:

- `components/explore/ResolvedImage.tsx`

Rules:

- uses `next/image`
- keeps aspect ratio through wrapper classes
- prevents layout breakage via `fill` + `object-cover`
- accepts ordered `sources`
- switches to the next source on image load failure

## Current Page Effect

After this phase:

- Explore inspiration cards display real runtime images
- featured archives display real cover images plus editorial badge
- archive feed cards display destination-aware or fallback cover images
- Explore home now reads like a real product surface instead of a text-only skeleton

2. decide whether image priority should be elevated for above-the-fold cards
